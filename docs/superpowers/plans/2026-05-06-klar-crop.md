# Plan: klar-crop & Avatar-Upload mit Crop

**Datum:** 2026-05-06
**Scope:** Generische `<klar-crop>`-Komponente + Integration in User-Avatar-Upload (Settings + Header)

## Motivation

Aktuell cropt `auth.service.uploadAvatar()` automatisch auf 128×128 zentriert — User kann
Bildausschnitt nicht steuern. Eine eigene `<klar-crop>`-Komponente erlaubt Pan/Zoom und
ist wiederverwendbar (künftig: Haushaltsmitglieder, Projekt-Cover etc.).

## Architektur

### `<klar-crop>` (apps/web/src/app/shared/ui/klar-crop.component.ts)

Inputs:
- `imageSrc: string` — DataURL oder ObjectURL
- `aspect: number = 1` — Crop-Ratio (Avatar = 1 = quadratisch)
- `outputSize: number = 256` — Kantenlänge des Output-Bildes
- `outputFormat: string = 'image/jpeg'`
- `quality: number = 0.85`
- `shape: 'circle' | 'rect' = 'circle'`

Outputs:
- `cropped` Event mit `{ dataUrl: string; blob: Blob }`

Public Method:
- `crop(): Promise<{ dataUrl, blob }>` — wird vom Wrapper aufgerufen

Implementation:
- Container 280×280 mit `overflow:hidden`
- Image mit `transform: translate(x, y) scale(s)`
- Pan via Pointer-Events (drag)
- Zoom via Wheel + Slider (auch Touch-Pinch via 2-Pointer-Tracking)
- Overlay zeigt Crop-Maske (Kreis bei `shape=circle`)
- minScale = 1.0 (cover), maxScale = 4.0
- Constraint: Image-Bounds dürfen Crop-Window nicht freigeben

### `<klar-image-crop-dialog>` (apps/web/src/app/shared/ui/klar-image-crop-dialog.component.ts)

Modal-Wrapper für Avatar-Upload-Flow:
- Inputs: `file: File`, `onConfirm: (dataUrl: string) => Promise<void>`, `outputSize?, shape?`
- Liest File als ObjectURL
- Rendert `<klar-crop>` + Action-Buttons (Speichern/Abbruch)
- Beim Speichern: ruft `crop()` auf, übergibt dataUrl an `onConfirm`, schließt Dialog

### Backend / Service

Backend bleibt unverändert (nimmt dataURL bereits an). Frontend-Service:
- `auth.service.uploadAvatarDataUrl(dataUrl: string): Observable<{ avatarUrl }>` — neue Methode
- Bestehende `uploadAvatar(file: File)` bleibt für Backwards-Compat erstmal, wird vom Header
  entfernt

### Integrationen

1. **`KlarHeaderUserComponent`**: bei File-Auswahl Dialog öffnen statt Auto-Crop
2. **Settings — Profil-Block**: neuer Eintrag "Profilfoto" mit 64px-Avatar-Preview + 
   "Foto ändern" und (falls gesetzt) "Foto entfernen"

## Tests

- Unit-Test: `klar-crop.spec.ts` — Constraint-Math (`clampOffset`)
- Unit-Test: Crop liefert Blob mit korrektem Typ und Größe
- Manuell: File hochladen → Dialog öffnet → Pan/Zoom → Speichern → Avatar aktualisiert
  in Header und Settings
